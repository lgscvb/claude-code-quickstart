# 從錯誤中學習

每次你遇到錯誤、bug、或是做錯事的時候，請執行以下步驟：

---

## 第一步：修好問題

先把問題修好，確認能正常運作。

## 第二步：檢討並記錄

修好之後，**一定要做這件事**：

找到專案根目錄的 `CLAUDE.md`（如果沒有就找 `~/.claude/CLAUDE.md`），在「踩過的坑」區塊加上一筆記錄。

格式：

```markdown
### [日期] 問題簡述
- **發生什麼事**：用一句話描述錯誤
- **根本原因**：為什麼會出錯
- **怎麼修的**：修復方法
- **以後怎麼避免**：下次遇到類似情況要注意什麼
```

範例：

```markdown
### 2026-03-04 忘記裝套件就直接 import
- **發生什麼事**：程式跑起來說找不到 `axios`
- **根本原因**：只寫了 `import axios`，但沒有先 `npm install axios`
- **怎麼修的**：執行 `npm install axios`
- **以後怎麼避免**：每次用新套件前，先確認 package.json 裡有沒有
```

## 第三步：回顧歷史教訓

每次開始新工作之前，先掃一眼 CLAUDE.md 裡的「踩過的坑」，確保不會重蹈覆轍。

---

## 重要原則

- **每次犯錯都是學習機會**，不要只是修完就算了
- **記錄要寫白話文**，讓完全沒技術背景的人也能看懂
- **如果 CLAUDE.md 裡沒有「踩過的坑」區塊**，就自己新增一個
- 這個機制讓 Claude 越用越聰明，因為它每次啟動都會讀 CLAUDE.md
